DBMS Introductie

Gegevens vs informatie

  • Gegevens zijn feiten/symbolen
  • Informatie zijn gegevens met praktisch nut of relevante nieuwswaarde

Voorbeeld

  • Telefoonboek bevat nummers met gekende gekoppelde gebruiker
  • Zonder naam van nummers = slechts gegevens
  • Dit kan zelfs als ruis gezien worden

Database

  • Opgeslagen gegevens = database
  • Manier gegevens zijn opgeslagen = datamodel
  • Software voor databases = DBMS

Wat is database

  • Een database is een (digitaal) archief
  • Ingericht om data flexibel te gebruiken
  • Databases worden overal gebruikt
  • Belangrijkste rol om data op te slaan

Minimale voorwaarden

  • Gegevens eenvoudig kunnen opslaan
  • Gegevens eenvoudig kunnen opzoeken
  • Gegevens kunnen gewijzigd worden
  • Gegevens verwijderen zoder dat het een nadelig effect heeft voor het systeem

Datamodel

Datamodel ookwel gegevensmodel of Bachmandiagram. = beschrijving van de structuur van gegens in informatiesysteem, welke gegevens en verbanden tussen gegevens.

  • tabellen
  • velden
  • sleutels
  • Relaties tussen de tabellen

DBMS

  • DBMS krijgt instructies, handelt nooit zelf
  • Kan manueel of met programma
  • DBMS word aangesproken door bepaalde talen
  • DBMS verwerkt de instructies
  • Belangrijkste databasetaal is SQL
  • Handhaven van integriteit van de gegevens

integriteit

  • Verschillende databases mogen elkaar niet tegenspreken (inconsistentie)
  • Redundantie moet tegengegaan worden (redundantie = dubbele data)

SQL

Databases en SQL

  • Niet zomaar wat data in database
  • Gegevens hebben structuur, eigenschappen en mogelijks relaties met andere gegevens
  • SQL statements worden op 2 manieren gebruikt
    • Ingetypt worden en direct uitgevoerd
    • Direct uitgevoerd door hogere generatietaal

DDL

  • Data Definition Language
  • Structuur aan database brengen
  • Essentieel om aan te beginnen

DCL

  • Data Control Language
  • Rechten toevoegen aan database
  • Uitgevoerd door administrator

DML

  • Data Manipulation Language
  • Verwerken van gegevens in de database
  • Word door gebruiker gebruikt

Relational datamodel

Bachmandiagram

  • beschrijving van structuur van gegevens
  • zichtbaar welke gegevens worden opgeslagen
  • weergeven van relaties tussen tabellen en gegevens

Tabellen, kolommen en rijen

  • Alle data wordt opgeslagen in tabellen
  • Officiele naam tabel is relatie
  • Een rij in een tabel is dus 1 record
  • Tabel heeft 2 belangrijke eigenschappen
    • Kruising rij en kolom kan maar 1 waarde bevatten
    • Rijen in tabel hebben geen specifieke volgorde

integriteit

  • Kwaliteitskenmerk van gegevens
  • Synoniem voor betrouwbaarheid
    • Juist (rechtmatigheid)
    • Volledig (niet te veel, niet te weinig)
    • Tijdig (op tijd)
    • Geautoriseerd (gemuteerd door een gerechtegd persoon)

Sleutels (keys)

  • Primary key (PK)
  • Candidate key (CK)
  • Alternative key (AK)
  • Foreign key (FK)

Primary key

  • Word gebruikt als unieke identificatie van record
  • Kan bestaan uit 1 of meerdere kolomment
  • 2 verschillende rijen mogen nooit dezelfde PK hebben

Candidate key

  • Soms meerdere mogelijkheden om PK te maken
  • Slechts 1 optie word effectief PK
  • Velden die de mogelijkheden hebben als PK worden dan CK's genoemd

Alternative key

  • De CK die niet de PK is
  • Verzamelnaam voor alle PK's en CK's

Foreign key

  • Populatie is deelverzameling van PK van een andere tabel
  • Gebruikt om relatie tussen tabellen vast te leggen

Relaties

Welke bestaan er

  • 1 op 1 relatie (1 op 1)
  • 1 op veel relatie (1 op n)
  • veel op 1 relatie (n op 1)
  • veel op veel relatie (n op n)
